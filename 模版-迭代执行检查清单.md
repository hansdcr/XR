# 迭代执行检查清单

## 使用说明

本检查清单用于确保每次迭代的执行质量。每完成一个迭代，按照以下清单逐项检查。

---

## 迭代开始前

### 准备工作
- [ ] 已完成前一个迭代
- [ ] 理解了前一个迭代的所有代码
- [ ] 代码已提交到版本控制（如使用 Git）
- [ ] 开发环境正常工作
- [ ] 阅读了本次迭代的目标和要求

### 理解迭代需求
- [ ] 清楚本次迭代的目标
- [ ] 理解要新建/修改哪些文件
- [ ] 知道要实现哪些方法
- [ ] 了解引入的新概念

---

## 迭代执行中

### 编码阶段
- [ ] 按照文档顺序实现代码
- [ ] 每完成一个方法就测试一次
- [ ] 保持代码简洁（方法 < 50行）
- [ ] 添加必要的注释
- [ ] 遵循命名规范

### 代码质量
- [ ] 没有语法错误
- [ ] 没有编译警告
- [ ] 代码格式整齐
- [ ] 变量命名清晰
- [ ] 逻辑结构清晰

### 增量测试
- [ ] 每写一段代码都能编译通过
- [ ] 及时发现和修复问题
- [ ] 不累积错误

---

## 迭代完成后

### 功能验证
- [ ] 按照"测试验证"部分的步骤测试
- [ ] 所有预期功能都正常工作
- [ ] 没有崩溃或异常行为
- [ ] UI 显示正确
- [ ] 控制台输出符合预期

### 代码理解
- [ ] 理解每一行代码的作用
- [ ] 知道为什么这样写
- [ ] 能够解释给他人听
- [ ] 理解本次迭代引入的新概念
- [ ] 能够独立重写这段代码

### 学习重点掌握
- [ ] 掌握了本次迭代列出的学习重点
- [ ] 理解了关键技术的原理
- [ ] 能够应用到其他场景

### 文档记录
- [ ] 记录遇到的问题和解决方案
- [ ] 标注不理解的地方（稍后询问）
- [ ] 更新个人学习笔记

### 代码整理
- [ ] 删除调试代码
- [ ] 删除未使用的导入
- [ ] 整理注释
- [ ] 提交代码到版本控制

---

## 问题处理

### 遇到错误时
- [ ] 仔细阅读错误信息
- [ ] 检查拼写错误
- [ ] 检查语法错误
- [ ] 对比文档示例代码
- [ ] 搜索相关错误信息
- [ ] 向 Claude 提问（提供具体错误信息）

### 不理解某个概念时
- [ ] 重读文档的"代码详解"部分
- [ ] 查阅官方文档
- [ ] 搜索相关教程
- [ ] 向 Claude 提问（具体说明哪里不理解）
- [ ] 尝试修改代码观察效果

### 功能不符合预期时
- [ ] 检查是否完全按照文档实现
- [ ] 检查是否有遗漏的步骤
- [ ] 对比参考代码
- [ ] 使用断点调试
- [ ] 添加日志输出

---

## 继续下一个迭代前

### 准备工作
- [ ] 本次迭代的所有检查项都已完成
- [ ] 代码可以正常运行
- [ ] 没有遗留问题
- [ ] 已做好学习笔记
- [ ] 精神状态良好，可以继续学习

### 节奏控制
- [ ] 不急于求成，确保质量
- [ ] 累了就休息，保持专注
- [ ] 必要时复习前面的迭代

---

## 阶段完成后

### 阶段回顾（每5-10个迭代）
- [ ] 回顾本阶段的所有迭代
- [ ] 理解整体架构
- [ ] 总结学到的知识
- [ ] 尝试从头重写部分代码
- [ ] 思考如何改进

### 知识巩固
- [ ] 能够画出应用的架构图
- [ ] 能够解释数据流
- [ ] 能够说明关键设计决策
- [ ] 理解为什么这样组织代码

---

## 结对编程检查

### 与 Claude 协作时
- [ ] 清楚地描述问题
- [ ] 提供具体的代码和错误信息
- [ ] 询问"为什么"而不只是"怎么做"
- [ ] 理解解释后再继续编码
- [ ] 讨论不同的实现方式

### 沟通质量
- [ ] 使用准确的技术术语
- [ ] 具体指出代码位置（文件名:行号）
- [ ] 描述预期行为 vs 实际行为
- [ ] 提供足够的上下文信息

---

## 快速检查清单（每次迭代必查）

核心 5 项：
1. [ ] **编译通过** - 没有错误和警告
2. [ ] **测试通过** - 功能符合预期
3. [ ] **代码理解** - 知道每行代码的作用
4. [ ] **概念掌握** - 理解新引入的技术点
5. [ ] **可以继续** - 本次迭代没有遗留问题

如果以上 5 项都打勾，可以安全地进入下一个迭代。

---

## 长期学习检查（每周/每月）

### 知识体系
- [ ] 定期总结学到的知识
- [ ] 建立自己的知识图谱
- [ ] 关联不同的技术概念
- [ ] 找出学习中的薄弱环节

### 实践应用
- [ ] 尝试独立完成小项目
- [ ] 修改示例代码实现新功能
- [ ] 解决实际问题
- [ ] 分享学习经验

### 持续改进
- [ ] 回顾学习效率
- [ ] 调整学习节奏
- [ ] 优化学习方法
- [ ] 设定新的学习目标

---

## 模版使用技巧

### 1. 打印或电子版
将本清单打印出来，或在笔记应用中创建电子版本，每次迭代都过一遍。

### 2. 个性化调整
根据自己的学习习惯，添加或删除某些检查项。

### 3. 追踪进度
- 使用 ✅ 标记已完成项
- 使用 ⚠️ 标记需要注意的项
- 使用 ❓ 标记不确定的项

### 4. 问题日志
建立一个单独的问题日志，记录：
- 遇到的错误
- 解决方案
- 花费的时间
- 学到的经验

### 5. 定期回顾
- 每周回顾本周的所有迭代
- 每月回顾本月的学习成果
- 总结常见问题和模式

```markdown
   - 解决方案: [方案]
   - 用时: X 分钟

### 学到的知识
- [知识点1]
- [知识点2]

### 需要进一步学习
- [待深入的概念]

### 笔记
[其他想记录的内容]
```

**版本**: 1.0
**更新日期**: [日期]
**用途**: 确保迭代开发的质量和学习效果

# 迭代开发模版系统 - 使用说明

## 模版概述

本模版系统提供了一套完整的结对编程和迭代开发流程，帮助你高效学习新技术并构建高质量的应用。

## 模版文件说明

### 1. 模版-项目规划总览.md
**用途**: 项目启动时使用，定义项目的整体规划和学习路径

**何时使用**:
- 开始一个新项目
- 学习一项新技术
- 需要系统化地掌握某个领域

**填写要点**:
- 明确项目目标和核心功能
- 分析技术栈和学习重点
- 将项目分解为多个阶段
- 估算每个阶段的迭代数量

### 2. 模版-迭代计划详解.md
**用途**: 详细规划每个迭代的实现步骤

**何时使用**:
- 完成项目规划后，开始编写具体迭代
- 每个迭代都按此模版编写

**填写要点**:
- 每个迭代目标明确且可测试
- 代码示例完整且可运行
- 包含详细的代码解释
- 提供测试验证步骤

### 3. 模版-迭代执行检查清单.md
**用途**: 执行迭代时的质量检查

**何时使用**:
- 每次开始迭代前
- 迭代进行中
- 迭代完成后

**使用方式**:
- 打印或在笔记应用中复制
- 逐项检查，确保质量
- 记录问题和解决方案

### 4. 本文档
**用途**: 说明如何使用整个模版系统

## 使用流程

### 阶段 1: 项目规划（1-2小时）

1. **复制模版文件**
   ```bash
   cp 模版-项目规划总览.md [你的项目名]-学习规划总览.md
   cp 模版-迭代计划详解.md [你的项目名]-迭代计划详解.md
   ```

2. **填写项目规划总览**
   - 定义项目目标
   - 分析技术栈
   - 规划学习阶段
   - 估算迭代数量（建议20-40个）

3. **与 Claude 讨论规划**
   - 让 Claude 审核规划的合理性
   - 调整迭代的粒度
   - 确保每个迭代 15-30 分钟完成

### 阶段 2: 编写迭代计划（3-5小时）

1. **按阶段编写迭代**
   - 从最简单的开始
   - 每个迭代独立可测试
   - 循序渐进增加复杂度

2. **遵循迭代模版规范**
   - 目标明确
   - 代码完整
   - 测试清晰
   - 解释详细

3. **与 Claude 协作优化**
   - 让 Claude 帮助拆分过大的迭代
   - 补充技术细节
   - 添加架构图和流程说明

### 阶段 3: 执行迭代（持续）

1. **每次迭代前**
   - 阅读迭代目标和要求
   - 检查开发环境
   - 确保理解前一个迭代

2. **执行迭代**
   - 按顺序实现代码
   - 边写边测试
   - 遇到问题及时提问

3. **完成迭代后**
   - 运行测试验证
   - 使用检查清单检查
   - 确保理解所有代码
   - 做好笔记

4. **阶段性回顾**
   - 每完成 5-10 个迭代回顾一次
   - 总结学到的知识
   - 巩固理解

**请 Claude 帮助**:

**遇到问题时**:
```

实际行为: [实际怎样]
```

**需要解释时**:
```

我的理解是 [你的理解]
能否详细解释一下原理？
```

### 4. 回顾阶段

**请 Claude 帮助回顾**:
```
我已完成迭代 1-10，请帮我：
1. 梳理学到的关键概念

3. 指出可能遗漏的知识点
4. 建议如何巩固学习
```

## 自定义模版

### 调整迭代粒度

**更小的迭代**（适合初学者）:
- 10-15 分钟完成
- 引入 1 个新概念
- 20-40 行代码

**更大的迭代**（适合有经验者）:
- 30-60 分钟完成
- 引入 2-3 个新概念
- 80-150 行代码

- 移动开发（iOS, Android, Flutter）

## 常见问题

### Q1: 迭代太多，如何管理？
**A**: 使用阶段划分，每个阶段 5-10 个迭代。完成一个阶段后休息和回顾。

### Q2: 某个迭代太难，无法完成怎么办？
**A**: 将其拆分为 2-3 个更小的迭代。没有迭代应该让你卡住超过 1 小时。

### Q3: 是否需要严格按照迭代顺序？
**A**: 建议严格按顺序，因为每个迭代都依赖前面的基础。但如果有充分理由，可以调整。

### Q4: 如何知道理解得足够深？
**A**: 使用"费曼技巧"：能够用简单的语言向他人解释，或者能够独立重写代码。

### Q5: 多久回顾一次？
**A**:
- 每个迭代后：快速回顾（5 分钟）
- 每 5 个迭代：阶段回顾（15 分钟）
- 每周：总体回顾（30 分钟）

### Q6: 代码和文档不一致怎么办？
**A**: 以实际运行的代码为准，同时更新文档。保持文档和代码同步很重要。

### Q7: 可以跳过某些迭代吗？
**A**: 如果你已经掌握了某个迭代涉及的技术，可以快速浏览并验证理解，然后继续。但建议至少运行一次代码。

## 进阶技巧

### 1. 创建个人知识库
将每个迭代学到的知识整理成：
- 概念卡片
- 代码片段库
- 最佳实践列表
- 常见错误和解决方案

### 2. 建立项目模版库
对于常见的项目类型，保存：
- 规划模版
- 迭代模版
- 代码模版
- 配置文件

### 3. 测量学习效率
记录每个迭代的：
- 计划时间 vs 实际时间
- 遇到的问题数量
- 理解程度评分
- 代码质量评分

### 4. 持续改进
定期回顾和优化：
- 迭代粒度是否合适
- 文档是否清晰
- 学习节奏是否舒适
- 知识保留率如何
